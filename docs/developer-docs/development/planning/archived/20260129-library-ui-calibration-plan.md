# 模板库 UI 与功能校准实施计划 (归档)

## 任务背景
在 2026-01-29 的迭代中，用户反馈模板库存在严重的视觉不一致问题，包括分类树指示条重复、高度无法撑满页面、表格样式陈旧且缺乏批量操作功能。

## 实施方案

### 1. 分类树 (CategoryTreeWidget)
- **校准内容**：彻底解决指示条重复。
- **技术点**：禁用原生 `Indentation`，采用自定义 `ItemWidget` 结合 `Spacer` 手动实现缩进。
- **视觉反馈**：解决背景叠加产生的阴影叠影 Bug。

### 2. 布局高度 (LibraryView)
- **校准内容**：组件垂直撑满。
- **技术点**：遍历并移除 `BasePageView` 默认添加的所有底部弹簧（Stretch），并为 `QSplitter` 设置 `Expanding` 政策。

### 3. 表格重构 (TemplateTableWidget)
- **校准内容**：回归原生标准，增加功能。
- **技术点**：
    - 基类切换为 `qfluentwidgets.TableWidget`。
    - 使用 `setBorderRadius(8)` 实现圆角高亮。
    - 增加复选框列（批量选择）与操作列（编辑/删除快捷按钮）。
    - 杜绝 QSS 硬编码色值。

## 验证结论
所有项均已完成，视觉对齐 JoJo 风格示例图，功能操作流畅。
