## 🎯 项目简介

Ghost-Dir 是一款专为 Windows 设计的**跨磁盘目录迁移与连接管理工具**。

### 核心价值

- **无缝迁移**：在任意磁盘间迁移文件夹，保持原路径有效
- **零数据丢失**：原子事务回滚机制，确保数据安全
- **智能管理**：自动扫描、模版库、批量操作，提升效率
- **现代 UI**：Fluent Design 设计语言，Windows 11/10 原生特效

### 典型应用场景

1. **系统盘空间不足**：将大型软件/游戏迁移到其他驱动器，原位置保留连接点
2. **软件重装**：保存常用软件配置为模版，快速重建环境
3. **多设备同步**：统一管理多台电脑的软件部署
4. **开发环境**：迁移 node_modules、.venv 等大型依赖目录

## 🔧 技术特性

### 架构设计

```
┌─────────────────────────────────────┐
│          GUI Layer (界面层)          │
│   Views / Dialogs / Components      │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│       Service Layer (服务层)        │
│  ConnectionService / TemplateService │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│        Data Layer (数据层)          │
│   TemplateManager / UserManager     │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼───────────────────┐
│        Core Layer (核心层)          │
│ Transaction / ProcessGuard / Scanner │
└─────────────────────────────────────┘
```

### 核心组件

| 组件 | 功能 | 特性 |
|------|------|------|
| `TransactionManager` | 事务管理 | 原子操作、自动回滚 |
| `ProcessGuard` | 进程卫士 | 文件占用检测、进程结束 |
| `SmartScanner` | 智能扫描 | 自动发现软件、模版匹配 |
| `TemplateManager` | 模版管理 | 官方库、自定义模版 |
| `ConnectionService` | 连接服务 | 业务逻辑封装 |

## 📦 安装与使用

### 系统要求

- **操作系统**：Windows 10/11
- **Python 版本**：3.8+（开发模式）
- **权限要求**：管理员权限
- **文件系统**：NTFS

### 快速开始

#### 方式一：下载可执行文件（推荐）

1. 前往 [Releases 页面](https://github.com/gzers/ghost-dir/releases/tag/v1.0.0)
2. 下载 `Ghost-Dir-1.0.0-win64.zip`
3. 解压后运行 `Ghost-Dir.exe`
4. 在 UAC 提示时点击"是"授予管理员权限

#### 方式二：从源码运行

```bash
# 克隆仓库
git clone https://github.com/gzers/ghost-dir.git
cd ghost-dir

# 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 运行程序
python run.py
```

### 使用流程

1. **首次启动**：程序自动请求管理员权限
2. **智能向导**：扫描本机软件，批量导入
3. **建立连接**：选择项目，点击"建立连接"
4. **享受空间**：源驱动器空间释放，软件正常运行

详细使用指南请参考 [用户文档](docs/user-docs/README.md)。

## 🐛 已知问题与限制

### 系统限制

1. **仅支持 Windows**：连接点（Junction）是 Windows 特有功能
2. **需要 NTFS**：目标驱动器必须是 NTFS 文件系统
3. **同一物理驱动器**：连接点不支持跨物理驱动器

### 功能限制

1. **需要管理员权限**：创建连接点需要提升权限
2. **不支持网络驱动器**：仅支持本地磁盘
3. **路径长度限制**：Windows 最大路径长度 260 字符
